SELF NOTES GIT BASIC COMMANDS

Git basic commands and blueprint to commit project from local to GitHub.
-Wilou 31/08/2022

Project commit for the first time

Select the folder and right click on it. Then click on "Git Bash Here". Once the terminal open, enter those commands:

  • $ git init: to initialize local git repository for the selected folder.
  • $ git add file.name: to add the selected file to index.
  • $ git add . : to add the all files to index.
  • $ git status: to check the status of the working tree. To use at each step of the commitment process to check file status.
  • $ git rm --cached file.name: to remove an added file.
  • $ git commit: it will open the text editor, un-comment the line "initialize..." and close. It will automatically commit.
  • $ git commit -m 'comment': to commit all files without opening the text editor.

To commit on remote repository

Once all files commited, next part will be about commiting on remote repository GitHub. First thing to do is to go on GitHub and create a new repository.

Follow GitHub blueprint which are the "â€¦or create a new repository on the command line" section, starting from the branch command:

  • $ git branch -M main: to go on the 'main' branch.
  • $ git remote add origin https://github-link.git: to link the remote repository with the local one.
  • $ git push -u origin main: to push the local repository to the remote one.

If someone is working on the same remote repository, and you need to get the updated file:

  • $ git pull: to pull remote repository to the local one. It will update the local files.

If you want to clone a repository:

  • $ git clone: to clone repository into a new directory.
